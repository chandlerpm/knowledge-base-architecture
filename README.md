## Design Philosophy

A well-designed knowledge base is not just a place to store information — it's a system for making information findable, trustworthy, and maintainable. This framework is built on four principles:

1. **Structure serves the user** — taxonomy is designed around how people look for information, not how the organization is structured internally
2. **Ownership is explicit** — every article has a named owner and a review date
3. **Content has a lifecycle** — articles that aren't maintained are more dangerous than no article at all
4. **Governance is lightweight** — enough process to ensure quality, not so much that contributors stop contributing
